Class Clone

This is the framework that implements the true prototype-based OOP paradigm in JS. It based on the new ECMA Script 5 features like Object.create and property descriptors.

Naming conventions

Var names, prefixed by "$", contain object, used as prototype for other objects. For example:
var $array = Array.prototype, $myType = {},
myTypeInstance = Clone($myType);

properties, prefixed by "_", are private.


Defined in: clone.js.

Clone(baseObj, properties, defaultDescriptor)
If called as function, return a clone of baseObj, and set it's own properties. If called by new operator, constructor will take 2 arguments (properties and defaultDescriptor).
Author: Alex Shvets. 
    var $myType = Clone.make({
        constructor: Clone.defineType('MyType', function(){
            this.applySuper(arguments);
            // do something with this...
        }),
        property: 1
    });
    var myTypeInstance = $myType.create();
    assert( $myType.isPrototypeOf(myTypeInstance) );

    var $myArray1 = Clone(Array.prototype, {customMethod: function(){}});
    var $myArray2 = Clone.makeFomClass(Array, {customMethod: function(){}});

    var myObj = {a:1, b:2, c:3};
    var cloneOfMyObj = Clone(myObj);
    cloneOfMyObj.a = 11; // myObj.a still == 1

    var myClone = new Clone();
    myClone.defineProperties({a:1, b:2, c:3});
